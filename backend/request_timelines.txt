to add a "lease" to a database
1) import backendlogic
    1.1) upload a lease via front-end + fastapi. process with pdfplumber, save to local and save pathname in next function
2) query_lease(pathname from upload, owner_email from front-end, raw_text from processed pdf) 
    2.1) this step comes after the pdf is uploaded and text is extracted via pdfplumber
    2.2) returns a lease_id. keep track of this in cookies (front end), use this to record the lease into the db after ai generation
3) query_response(response from llm (see ds1.1), lease_id = lease_id from cookies, owner_id lookup, or saved call)
    3.1) return value: lease_id if success, error if fail
    3.2) implement error handling in case of mishap


to add a set of translations to a database
1) retrieve translation info from front end
    1.1) includes: lease_id, language (2-letter/4-letter code, mapped to dictionary)
2) use GEMINI to translate via prompt
    2.1) pass in ds1.2 to ensure consistency in translation
    2.2) implement error handling in case of AI misunderstanding
3) add into database via add_translations(lease_id from frontend, translations from GEMINI, language_code (2/4 digit from frontend))
    3.1) important enough to restate: make sure to add error handling to handle occasional AI mishaps
    3.2) returns # of translations added if success, error if fail

to pull a lease from the database
1) retrieve lease id from owner lookup with lease_id_by_owner(owner_id = owner_email from frontend)
2) pull_report_data(lease_id from prior call)
    2.1) returns lease data in form of ds1.3, use front end to parse into actionable text
    2.2) returns if true, if error, exception will be called

to pull translations from the database
1) recieve lease_id (or owner_id) from frontend, as well as translation_language_code
2) if translation_language_code not found in owner's translations, go add (see part 2)
3) if found, get_translated_report_data(lease_id from owner_lookup or frontend, language_code from front end)
    3.1) should return data in form of ds1.3, but replaces english text with custom language text
4) send back to front-end
    4.1) set front end flag to indicate text is in another language. update elevenlabs as result
